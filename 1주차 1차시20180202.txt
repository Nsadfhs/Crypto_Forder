네트워크의 주의점

1.컴퓨터 네트워크망을 통하려 전송된 자료가 전송자 수신자 사이에 다른사람들에게 '노출'되지 않았는지

2.전송된 자료가 누군가에 의해 '변형'되지 않았는지

3.송신자가 용의자가 아닌지 - '스팸'

4.자료를 보내지 않고 보냈다고 주장하지 않는지/ 자료를 받아놓고 못 받았다고 하지는 않는지 - '부인(不認)방지'

5.사이트에 공격자가 간섭하지 않았는지

6.개인정보보안에 결점이 없는지(아이디 패스워드 인증 서명)

7.통신의 중간에 누군가가 간섭하지는 않는지. - 끼어들기(도청), 가로채기(도용)

보안 / 보호

Rand Report R-609보고서(보안개념의 변화계기)
보안문제 : '데이터'보안, '데이터접근제한', '인적구성원(공격자)'에 대한 보안

네트워크에 연결된 노드 수의 폭발적증가(1970) ->보안에 대한 심각한 문제가 발생 -> 암호를 이용 ->대칭암호 -> 공개키암호(유지보수가 쉬움)

1980년 : 
1. PC업무의 네트워크화
2. TCP(파일 전송 프로토콜)/IP(인터넷 프로토콜) - 인터넷 기초 환경 구축
3.금융기관, 관공서등 민감한 자료를 보관하고 처리하는 곳 역시 심각한 보안문제

1990년:

1.www웹브라우저개발(95년 i.e개발)
2.인터넷 표준화가 아직 이루어지지 않았음
3.초기에는 물리적 보안뿐 -아직까지 보안위협에 많이 노출

2000년:

1.네트워크와 인터넷의 가장 큰 역활중 하나 - 금융거래 방식과 구매방식에 대혁명이 일어남
2.전자상거래 활성에 따른 보안문제
3.이동통신 정보전달에 따른 고속화에 따른 이동통신보안문제 또한 위치정보에 대한 정보도 문제가 됨.

2010년~현재 :

1.무선통신의 발달로 보안문제가 또다시 대두
2.유선망의 정보보호나 보안문제보다 기술과 방법에서 새로운 개념으로 인한 무선망에서 더 많은 위협이 대두됨
3.개인들 또한 정보보호나 프라이버시등에 대한 개념이 확산. 

-과연 내 컴퓨터는 안전한가? 내 컴퓨터와 연결된 모든 컴퓨터의 보안수준에 따라 상이하게 다르다. 즉, 안전하지 않다.

*보안 : 가치있는 유형과 무형의 자산을 도난, 소실, 유출로 부터 보호하는 것
*보호 : 보안보다 넓은 의미를 가지고 있으며, 
1.위협으로부터 안전한 정도
2.정보를 저장하거나 유통하는 전반적인 시스템의 안정성

보안

-물리적 보안 : 물리적인 대상, 영역을 보호하는 것
-인적 보안 : 개인 혹은 집단을 보호하는 것
-운용보안 : 특정운용이나 일련의 동작의 세밀한 사항을 보호하는 것
-통신보안 : 통신미디어, 기술, 내용을 보호하는 것
-네트워크보호 : 네트워킹 요소, 연결상태, 내용등을 보호하는 것
-정보보호 : 개인이나 조직등의 정보자산을 보호하는 것

정보보호 시스템(정보보호 정책) : 

1.정보보호관리
2.컴퓨터및 데이터 보관
3.네트워크 보안

정보보안의 3대 요소

2.무결성 : 변조 파괴 방지
3.가용성 : 허가부여
